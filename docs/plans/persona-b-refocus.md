# MyLifeOS — 페르소나 B 리포커싱 개발 지시

## 배경 및 목표

현재 MyLifeOS는 F-01~F-45, 45개 기능이 구현된 상태다.
이번 작업의 목표는 **"데이터 기반 자기계발을 원하는 사람"(페르소나 B)** 에 집중하도록
기능을 정리하고, AI 인사이트가 서비스의 핵심으로 느껴지도록 구조를 재편하는 것이다.

**기준 원칙:**
> "이 기능이 AI 인사이트의 원료가 되는가?"
> - YES → 유지
> - NO → 제거 또는 숨김 처리

---

## 1. 제거할 기능

아래 기능들은 AI 인사이트와 무관한 독립 유틸리티다.
관련 페이지, 컴포넌트, API Route, DB 마이그레이션, 네비게이션 항목을 모두 제거한다.

| 기능 | 이유 |
|------|------|
| F-36 투자 트래킹 (`/money/investments`, `investments-tab.tsx`) | 전용 앱 대체 가능, AI 인사이트 원료 아님 |
| F-45 장보기 목록 (`/time/shopping` 또는 `/money/shopping`) | 순수 유틸리티 |
| F-42 독서 기록 (`/books`) | 생활 패턴 데이터와 교차 분석 불가 |
| F-39 포모도로 타이머 (`/time/pomodoro`) | 전용 앱 대체 가능, 세션 데이터 미활용 |
| F-41 약 복용 기록 (`/health/medications`) | 의료 데이터 책임 문제, 일반 패턴 분석 부적합 |
| F-27 인간관계 메모 (`/private/relations`) | 암호화 저장은 좋으나 AI 분석 불가 |
| F-40 퀵 메모 (`/time` 또는 별도 경로) | 순수 유틸리티 |

**제거 작업 범위 (각 기능별 동일하게 적용):**
- `app/(dashboard)/[경로]/` 페이지 디렉토리 삭제
- `components/[모듈]/` 관련 컴포넌트 삭제
- `app/api/[엔드포인트]/` Route Handler 삭제
- `hooks/use-[기능].ts` 훅 삭제
- 사이드바(`components/common/sidebar.tsx`), 바텀 네비(`components/common/bottom-nav.tsx`), 헤더 네비에서 해당 항목 제거
- `messages/ko.json`, `messages/en.json`에서 관련 번역 키 제거
- 해당 기능으로의 redirect가 있다면 함께 제거

## 2. 유지 및 단순화할 기능

### 유지 (핵심 데이터 원료)

| 도메인 | 유지 기능 | 비고 |
|--------|-----------|------|
| 시간 | 루틴, 캘린더/할일, 스트릭 | 습관 달성률 |
| 머니 | 지출/거래, 예산, 정기지출 | 소비 패턴 |
| 건강 | 수면, 운동, 식사, 체중/체성분 | 핵심 4대 지표 |
| 건강 | 음주 기록 | 건강 섹션에 통합 유지 |
| 기분 | 일기, 감정 태그 | 주관적 컨디션 교차 분석 |
| 자산 | 자산 현황 | 재무 맥락 (단순 유지) |
| 목표 | F-34 장기 목표 | 진척도 → AI 인사이트 연결 |

### 단순화

- **수분 섭취 (F-38):** 별도 섹션 대신 건강 홈 페이지에 원탭 로깅 위젯으로 축소
- **자산 현황 (F-21):** 투자 탭 제거 후 자산 파이차트 + 월별 추이만 유지

## 3. AI 인사이트 우선 노출 (핵심 UX 변경)

현재 AI 인사이트는 `/reports` → "AI" 탭 (5번째)에 위치한다.
페르소나 B에서 AI 인사이트는 서비스의 핵심이므로 **홈 대시보드에 최우선 노출**한다.

### 변경 사항: `app/(dashboard)/page.tsx` (홈 대시보드)

현재 구조:
```
홈: 요약 카드 4개 (시간/머니/건강/프라이빗) + FAB
```

변경 후 구조:
```
홈:
1. AI 인사이트 섹션 (최상단)
   - 최신 인사이트 카드 2~3개 미리보기
   - "전체 보기" → /reports?tab=ai 링크
2. 오늘의 요약 카드 (기존 4개 카드, 아래로 이동)
3. FAB (기존 유지)
```

- `components/analytics/ai-insights-preview.tsx` 신규 컴포넌트 생성
- 최근 `ai_insights` 테이블에서 최신 3개 인사이트를 React Query로 조회
- 인사이트가 없는 경우: "아직 분석 데이터가 부족합니다. 일주일간 기록해보세요." 안내 문구 표시
- 인사이트 type별 색상 분기는 기존 `/reports` AI 탭 로직 재사용

## 4. 네비게이션 정리

제거된 기능에 따라 네비게이션을 아래와 같이 정리한다.

**바텀 네비 / 사이드바 (6개 유지):**
```
홈 | 시간 | 머니 | 건강 | 프라이빗 | 분석
```
- 투자, 목표, AI인사이트 독립 항목은 이미 제거된 상태 (e0e2465 커밋에서 처리됨) → 확인 후 중복 제거만 수행

**각 모듈 내부 탭/서브메뉴에서도 제거된 기능 항목 삭제**

## 5. 작업 순서 권장

1. 제거 대상 7개 기능 삭제 (페이지 → 컴포넌트 → API → 훅 → 네비 → 번역키 순)
2. `npm run type-check` 및 `npm run build`로 잔존 참조 오류 확인 및 수정
3. 홈 대시보드 AI 인사이트 우선 노출 구현
4. 수분 섭취 위젯 단순화
5. `npm test`로 전체 테스트 통과 확인
6. 제거된 기능 관련 E2E 테스트 삭제 또는 skip 처리
