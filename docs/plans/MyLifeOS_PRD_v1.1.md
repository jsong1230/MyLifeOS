## 1. 제품 개요

### 1.1 제품 비전

My Life OS는 단편적으로 관리되던 개인의 시간, 금전, 건강, 사적 기록을 단 하나의 플랫폼에서 통합 관리하는 개인용 라이프 매니지먼트 웹 애플리케이션이다. 가족과 지인 누구나 독립 계정으로 가입하여 자신만의 데이터를 완전히 분리된 환경에서 관리할 수 있다. iOS Safari의 '홈 화면에 추가' 기능을 통해 별도 앱 설치 없이 네이티브 앱과 동일한 사용 경험을 제공한다.

### 1.2 핵심 가치 제안

- **통합성**: 4개 핵심 영역(시간/금전/건강/사적 기록)을 하나의 대시보드에서 관리
- **멀티 유저**: 가족/지인 누구나 독립 계정 생성 → 데이터 완전 분리
- **앱 설치 불필요**: iOS Safari '홈 화면에 추가' → 네이티브 앱과 동일한 UX (PWA)
- **프라이버시 우선**: 민감한 개인 데이터를 완전히 본인만 접근 가능한 환경에서 관리
- **인사이트 제공**: 단순 기록을 넘어 패턴 분석 및 주간/월간 리포트 제공
- **마찰 최소화**: 빠른 데이터 입력 UX로 기록 습관 형성 지원

### 1.3 대상 사용자

**Primary User**: 가족 및 지인 포함 성인 누구나 (독립 계정 방식, 데이터 완전 분리)

| 항목 | 내용 |
|------|------|
| 나이/성별 | 성인 누구나 (연령 제한 없음, 가족 및 지인 포함) |
| 기술 수준 | 스마트폰 사용 가능한 누구나. iOS Safari 홈 화면 추가로 앱처럼 사용 |
| Pain Point | 일정·지출·건강·감정 기록이 여러 앱에 분산되어 있어 전체 흐름 파악이 어려움 |
| Goal | 내 삶의 모든 데이터를 한 곳에서 보고, 패턴을 발견하고, 더 나은 선택을 하고 싶음 |

## 2. 기술 스택 및 아키텍처

### 2.1 기술 스택

| 레이어 | 기술 | 선정 이유 |
|--------|------|-----------|
| Frontend | Next.js 14 (App Router) + TypeScript | 풀스택 지원, SEO, PWA 최적화 |
| Styling | Tailwind CSS + shadcn/ui | 빠른 UI 개발, 일관된 디자인 시스템 |
| Database | Supabase (PostgreSQL) | 인증+DB+실시간 올인원, 무료 플랜 |
| 인증 | Supabase Auth (이메일 + OAuth) | 소셜 로그인, RLS 보안 내장 |
| 차트/시각화 | Recharts | React 친화적, 가벼운 번들 사이즈 |
| 상태 관리 | Zustand + React Query | 서버/클라이언트 상태 분리 |
| 배포 | Vercel | Next.js 최적화, 자동 CI/CD |
| 암호화 | crypto-js (클라이언트 사이드) | 일기/사적 기록 추가 암호화 |

### 2.2 아키텍처 개요

- **클라이언트**: Next.js App Router, PWA manifest 설정으로 모바일 홈 화면 추가 지원
- **인증 레이어**: Supabase Auth → JWT 기반 세션 관리
- **데이터 레이어**: Supabase PostgreSQL + Row Level Security (RLS) 적용 — 본인 데이터만 접근 가능
- **암호화 레이어**: `diary_entries` 및 `relationship_notes` 테이블은 저장 전 클라이언트 사이드 AES 암호화 적용
- **API 레이어**: Next.js Route Handlers (서버 컴포넌트 + API Routes 혼용)

## 3. 데이터베이스 스키마

### 3.1 핵심 테이블 구조

| 테이블명 | 역할 | 주요 컬럼 |
|----------|------|-----------|
| `users` | 사용자 프로필 | id, email, name, timezone, pin_hash, created_at |
| `todos` | 할일 관리 | id, user_id, title, due_date, priority, status, category |
| `routines` | 반복 루틴 | id, user_id, title, frequency, time_of_day, streak |
| `routine_logs` | 루틴 실행 기록 | id, routine_id, date, completed |
| `transactions` | 수입/지출 | id, user_id, amount, type, category, memo, date |
| `budgets` | 월별 예산 | id, user_id, category, amount, year_month |
| `health_logs` | 건강 통합 기록 | id, user_id, log_type, value, unit, note, date |
| `meal_logs` | 식단 기록 | id, user_id, meal_type, food_name, calories, date |
| `drink_logs` | 음주 기록 | id, user_id, drink_type, amount_ml, drink_count, date |
| `diary_entries` | 일기 (암호화) | id, user_id, content_encrypted, mood, tags, date |
| `relationships` | 인간관계 메모 | id, user_id, person_name, last_met, notes_encrypted |

### 3.2 보안 정책 (RLS)

- 모든 테이블에 Row Level Security 활성화
- SELECT / INSERT / UPDATE / DELETE 정책: `auth.uid() = user_id` 조건 적용
- `diary_entries`, `relationships` 테이블: 클라이언트 사이드 AES-256 암호화 후 저장
- PIN 잠금: 앱 진입 또는 사적 기록 섹션 접근 시 PIN 검증 레이어 추가

## 5. 비기능 요구사항

### 5.1 성능

- 초기 페이지 로드: 3초 이내 (LCP 기준)
- 데이터 입력 후 UI 반영: 500ms 이내
- 대시보드 데이터 조회: 1초 이내

### 5.2 보안

- HTTPS 전용 서비스 운영
- Supabase RLS 전체 테이블 적용 — 본인 데이터만 접근 가능
- `diary_entries`, `relationships`: AES-256 클라이언트 암호화
- 세션 타임아웃: 30분 비활동 시 자동 잠금
- PIN 잠금: 최대 5회 시도 실패 시 10분 차단

### 5.3 사용성

- 모바일 우선(Mobile-First) 반응형 디자인
- 핵심 데이터 입력: 탭 3회 이내 도달 가능한 UX
- 오프라인 상태: 마지막 동기화 데이터 캐시 표시

### 5.4 확장성

- 모듈 단위 컴포넌트 설계로 추후 기능 추가 용이
- Supabase 무료 플랜 한도: DB 500MB, API 요청 50만회/월 — 소규모(가족/지인) 멀티 유저 기준 충분. 사용자 증가 시 Pro 플랜($25/월) 업그레이드 고려

## 6. 개발 로드맵

### Claude Code 기반 개발 순서

| Phase | 목표 | 주요 작업 | 예상 기간 |
|-------|------|-----------|-----------|
| Phase 0 | 환경 세팅 | Next.js + TypeScript + Tailwind + Supabase 초기 설정, 인증 구현, DB 스키마 생성 | 1-2일 |
| Phase 1 | 대시보드 & 기반 UI | 레이아웃, 네비게이션, 메인 대시보드 (4개 모듈 요약 카드) | 2-3일 |
| Phase 2 | 시간 관리 모듈 | Todo CRUD, 캘린더 뷰, 루틴 설정 및 체크인, 완료율 차트 | 3-4일 |
| Phase 3 | 금전 관리 모듈 | 수입/지출 CRUD, 예산 설정, 카테고리 관리, 지출 차트 | 3-4일 |
| Phase 4 | 건강 트래킹 모듈 | 식단/음주/수면/운동 기록, 건강 대시보드, 음주 경고 | 4-5일 |
| Phase 5 | 사적 기록 모듈 | 일기 에디터, 감정 캘린더, 관계 메모, 암호화 + PIN 잠금 | 4-5일 |
| Phase 6 | 통합 리포트 & 마무리 | 주간/월간 통합 리포트, PWA 설정, 성능 최적화, 배포 | 3-4일 |

### 우선순위 매트릭스

- **P0 (Must Have)**: 모든 모듈의 기본 CRUD, 대시보드, 인증, PIN 잠금, PWA 설정
- **P1 (Should Have)**: 통계 차트, 예산 경고, 자산 현황, 루틴 streak, 음주 경고
- **P2 (Nice to Have)**: 다크 모드, 타임블록, 정기지출, 감정 통계, 데이터 내보내기

```bash
npx create-next-app@latest my-life-os --typescript --tailwind --app
```

### 7.2 핵심 디렉토리 구조

```
my-life-os/
├── app/
│   ├── (auth)/          # 로그인, 회원가입 페이지
│   ├── (dashboard)/     # 메인 대시보드
│   ├── time/            # 시간 관리 모듈
│   ├── money/           # 금전 관리 모듈
│   ├── health/          # 건강 트래킹 모듈
│   └── private/         # 사적 기록 모듈 (PIN 미들웨어 적용)
├── components/
│   ├── ui/              # shadcn/ui 기반 공통 컴포넌트
│   └── modules/         # 각 모듈별 컴포넌트
├── lib/
│   ├── supabase/        # Supabase 클라이언트 및 DB 쿼리 함수
│   └── crypto/          # 암호화/복호화 유틸리티
└── store/               # Zustand 전역 상태
```

### 7.3 Claude Code 작업 지시 예시

```
Phase 0: Supabase 클라이언트 설정 및 /lib/supabase/client.ts 파일 생성. 환경변수 처리 포함.

사적 기록 모듈에 PIN 미들웨어 추가. /app/private 경로 접근 시 PIN 검증 로직 구현.
```
