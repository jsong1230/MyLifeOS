## 보안

### HIGH - salt 값 API 응답 노출

**파일:** `app/api/users/pin/route.ts:154` / `app/api/users/pin/verify/route.ts:83`

```
# route.ts:154
return NextResponse.json({ success: true, data: { pinSet: true, salt } })

# verify/route.ts:83
return NextResponse.json({ success: true, data: { verified: true, salt: pin_salt } })
```

**이슈:** bcrypt salt 값이 API 응답 바디에 포함되어 클라이언트에 그대로 노출됨.
bcrypt는 해시에 salt를 이미 내장하므로 별도 salt 반환은 불필요함.
salt를 알면 오프라인 사전공격 비용이 줄어드는 보안 위험이 존재함.

**권장 수정:** 응답에서 `salt` 필드를 제거할 것. 클라이언트가 PBKDF2 키 파생에 salt가 필요한 경우, salt를 서버에서 관리하거나 salt의 역할을 재설계할 것.

### WARNING - .env.local.example에 SUPABASE_SERVICE_ROLE_KEY 누락

**파일:** `.env.local.example`

```
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

**이슈:** `lib/supabase/admin.ts`에서 `process.env.SUPABASE_SERVICE_ROLE_KEY`를 사용하지만
`.env.local.example`에 해당 환경변수가 정의되어 있지 않음.
신규 개발자가 환경 설정 시 service_role key 설정을 누락하면 PIN API 전체가 런타임 오류 발생.

**권장 수정:** `.env.local.example`에 `SUPABASE_SERVICE_ROLE_KEY=your_service_role_key` 항목 추가.

### WARNING - PIN 검증 API rate limiting 미구현

**파일:** `app/api/users/pin/verify/route.ts`

**이슈:** 서버 사이드 계정 잠금 로직(5회 실패 → 10분 잠금)은 구현되어 있으나,
인증되지 않은 요청에 대한 IP 기반 rate limiting이 없음.
JWT 인증 통과 후에만 PIN 검증이 가능하므로 현재 구조에서는 위험이 제한적이나,
동일 계정으로 다중 병렬 요청 시 race condition으로 잠금 카운터가 정확하지 않을 수 있음.

**권장 수정:** DB 수준에서 atomic increment 사용 또는 Supabase Row Level Lock 검토.

## 성능

### WARNING - todosByDate에 useCallback 오용

**파일:** `components/time/calendar-view.tsx:58-61`

```tsx
const todosByDate = useCallback(
  (date: string) => todos.filter((todo) => todo.due_date === date),
  [todos]
)
```

**이슈:** `useCallback`은 함수 참조를 메모이제이션하지만, 내부 `filter` 연산 자체는 매번 실행됨.
날짜별 할일 맵을 `useMemo`로 미리 구성하지 않으면 MonthView 렌더링 시 42개 날짜 각각에 대해
O(n) filter를 반복 호출함. todos가 많아질수록 O(42 * n) 비용 발생.

**권장 수정:** `useMemo`로 `{ [date]: Todo[] }` 맵을 사전 구성하여 O(n) 전처리 + O(1) 조회로 변경.

### WARNING - monthlyTrendChart 로케일 하드코딩

**파일:** `components/money/monthly-trend-chart.tsx:34`

```tsx
return new Intl.DateTimeFormat(locale === 'ko' ? 'ko-KR' : 'en-US', { month: 'short' }).format(date)
```

**이슈:** `locale` 값이 'ko' 또는 'en'만 처리하고 그 외 locale을 'en-US'로 강제 변환함.
`useLocale()`로 실제 locale을 가져오고 있으므로, 직접 Intl에 전달하는 것이 더 견고함.
현재 지원 locale이 ko/en 두 가지뿐이라 실제 버그는 없으나, 확장성 부족.

**권장 수정:** `locale === 'ko' ? 'ko-KR' : 'en-US'` 대신 locale 매핑 객체 또는
`locale` 값을 그대로 `Intl.DateTimeFormat`에 전달하는 방식 검토.

### WARNING - recurring-summary.tsx 연간 총액 계산 로직 불일치

**파일:** `components/money/recurring-summary.tsx:31-35`

```tsx
// 연간 총액 = monthly * 12 + yearly 합산
const yearlyTotal = activeExpenses.reduce((sum, expense) => {
  return sum + expense.amount
}, 0)
```

**이슈:** 주석에 "monthly * 12 + yearly 합산"이라고 명시되어 있으나

**권장 수정:** 주석을 구현에 맞게 수정하거나, 의도가 "연 총 지출"이라면
`monthly * 12 + yearly`로 계산 로직을 수정할 것.
